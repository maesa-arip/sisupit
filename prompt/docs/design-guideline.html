Panduan desain untuk keadaan darurat.

Satu sistem yang menyatukan warna, tipografi, sudut, dan jarak Sisupit — di mode terang maupun gelap. Semua nilai memetakan token nyata di resources/css/app.css & tailwind.config.js. Aturan tunggal: gunakan token, jangan warna mentah.

01Prinsip Inti

Token dulu, selalu.

Audit menemukan pola lama: warna ditulis mentah untuk mode terang (bg-red-600) lalu token untuk gelap (dark:bg-destructive). Akibatnya satu merah bisa tampil tiga nuansa berbeda. Sistem ini menghapus itu.

Pakai token semantik. bg-destructive, text-success, border-info. Otomatis benar di light & dark, dan seragam dengan logo.

Jangan warna mentah. bg-red-600, text-blue-500, hex baru. Memecah brand & merusak dark mode.

Pengecualian sah: nilai warna pada marker peta Leaflet dan chart — keduanya butuh warna nyata, ambil dari token via JS bila memungkinkan.

03Status Laporan

Merah menyala → biru memadamkan.

Lifecycle: Masuk → Terverifikasi → Penanganan → Selesai (plus Ditolak). Warna menyandikan progres, dengan satu keputusan domain penting di ujungnya.

Masuk→ Terverifikasi→ Penanganan→ Selesai· Ditolak

Selesai = biru, bukan hijau. Ini aplikasi pemadam kebakaran — biru identik dengan air yang memadamkan api. Merah (darurat) di awal, biru (padam/aman) di akhir. Emerald dipakai untuk fase Penanganan yang aktif berjalan, dan Ditolak memudar jadi netral.

05Sudut / Rounded

Skala jari-jari dari satu sumber.

Basis --radius: 0.5rem (8px). Tailwind menurunkan sm/md/lg darinya; xl ke atas nilai tetap. Konsisten: makin besar & penting elemennya, makin besar sudutnya.

06Spasi & Layout

Kelipatan 4px, jarak lewat gap.

Skala spasi Tailwind (basis 4px). Atur jarak antar-elemen dengan flex/grid + gap, bukan margin per-elemen yang mudah dobel/kolaps.

2 · 8px — jarak ikon↔teks, gap chip
3 · 12px — padding tombol, gap kecil
4 · 16px — gap kartu di grid, padding rapat
6 · 24px — padding kartu/panel
12 · 48px — jarak antar-blok
16–18 · 64–72px — jarak antar-section

Lebar konten maksimum halaman: max-w-1000/1080px, dipusatkan (mx-auto) dengan padding tepi 24px.

12Do & Don't

Ringkasan aturan.

Lakukan
  • Pakai token: bg-destructive, text-success, border-info.
  • Bedakan penekanan lewat solid / outline / soft.
  • Selesai = info (biru/air), Penanganan = success.
  • Hidran = teal, relawan = volunteer.
  • Rounded naik sesuai bobot elemen; jarak lewat gap.
  • Uji setiap layar di mode terang dan gelap.
Hindari
  • Warna mentah baru: bg-red-600, text-blue-500, hex.
  • Pola light-mentah / dark:token — sumber tiga-nuansa-merah.
  • Dua merah mirip untuk brand vs bahaya.
  • Hijau 142 murahan untuk “aktif”.
  • Bayangan berat di elemen datar.
  • Mencampur dua icon library dalam satu komponen.
